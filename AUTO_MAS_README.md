# FGO自动化脚本 - AUTO-MAS集成说明

## 概述

本脚本已适配AUTO-MAS调度系统，可以与AUTO-MAS配合使用，实现自动化的FGO游戏操作。

## 文件说明

- `fgo_bot_auto_mas.py` - AUTO-MAS适配版脚本
- `run_auto_mas.bat` - AUTO-MAS启动脚本
- `config.ini` - 配置文件（与原版共用）

## AUTO-MAS配置指南

### 1. 脚本根目录
设置为项目根目录：`D:\autobgo\fgo-auto-login-bilibili`

### 2. 脚本路径
设置为：`run_auto_mas.bat`

### 3. 脚本启动参数
格式：`--device {ADB设备地址}`

示例：
- `--device 127.0.0.1:16416` (MuMu模拟器)
- `--device 127.0.0.1:5555` (雷电模拟器)
- `--device 127.0.0.1:62001` (夜神模拟器)

### 4. 脚本配置文件路径
设置为：`config.ini`

### 5. 脚本日志文件路径
设置为：`fgo_bot_log.txt`

### 6. 任务成功日志
设置为：`任务成功` 或 `[AUTO-MAS] 任务成功`

### 7. 任务异常日志
设置为：`任务失败` 或 `[AUTO-MAS] 任务失败`

### 8. 追踪脚本子进程
设置为：`关闭`

## 工作流程

AUTO-MAS集成后的工作流程：

1. **AUTO-MAS启动模拟器** - 由AUTO-MAS管理
2. **AUTO-MAS运行脚本** - 调用`run_auto_mas.bat --device {设备地址}`
3. **脚本连接ADB** - 使用指定的设备地址连接
4. **启动FGO游戏** - 通过ADB命令启动
5. **执行自动操作** - 按照配置执行点击和按键操作
6. **关闭FGO游戏** - 通过ADB命令停止
7. **脚本退出** - AUTO-MAS检测到脚本完成
8. **AUTO-MAS关闭模拟器** - 由AUTO-MAS管理

## 配置文件说明

### config.ini 关键配置

```ini
[Emulator]
; ADB设备地址（AUTO-MAS会通过命令行参数覆盖此设置）
ip_port = 127.0.0.1:16416

[Delays]
; 启动游戏后等待时间（秒）
post_launch = 30

; 步骤间基础间隔（秒）
step_base = 5

; 点击间隔（多次点击时，点击之间的延迟）
click_interval = 1

; 进入游戏点击与登录点击之间的间隔（秒）
enter_login_interval = 20

[Coordinates]
; 各个步骤的坐标配置（保持与原版相同）
tap_enter_game = 640,360,1
tap_login = 533,133,1
tap_back = KEYCODE_BACK,1
; ... 其他配置
```

## 命令行使用

### 直接运行（测试用）
```bash
run_auto_mas.bat --device 127.0.0.1:16416
```

### 通过AUTO-MAS运行
AUTO-MAS会自动调用脚本并传递设备地址参数。

## 日志说明

脚本会生成`fgo_bot_log.txt`日志文件，AUTO-MAS通过以下内容判断任务状态：

- **成功**：日志中包含`[AUTO-MAS] 任务成功`
- **失败**：日志中包含`[AUTO-MAS] 任务失败`

## 注意事项

1. **模拟器管理**：模拟器的启动和关闭完全由AUTO-MAS管理，脚本不再处理
2. **ADB连接**：脚本使用AUTO-MAS传递的设备地址进行连接
3. **坐标配置**：保持与原版相同的坐标配置（1280x720）
4. **时间间隔**：所有时间间隔配置保持不变
5. **错误处理**：脚本会在发生错误时输出`[AUTO-MAS] 任务失败`

## 常见问题

### Q: 如何确定ADB设备地址？
A: 不同模拟器的默认端口：
- MuMu模拟器：16416 + 索引（索引1=16416，索引2=16417）
- 雷电模拟器：5555 + 索引（索引0=5555，索引1=5556）
- 夜神模拟器：62001 + 索引（索引0=62001，索引1=62025）
- 蓝叠模拟器：5555

### Q: 脚本无法连接ADB？
A: 检查以下几点：
1. 模拟器是否已启动
2. ADB设备地址是否正确
3. 模拟器ADB调试是否开启
4. 防火墙是否阻止连接

### Q: AUTO-MAS无法检测任务完成？
A: 检查以下几点：
1. 日志文件路径配置是否正确
2. 任务成功/失败日志配置是否正确
3. 脚本是否正常退出
4. 追踪脚本子进程设置是否正确
